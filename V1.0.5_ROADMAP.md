## 1. 需求背景

### 1.1 需求变更原因

原计划的 **Script Mode**（自定义 JavaScript 脚本生成消息）存在以下问题：
- QuickJS WASM 集成复杂，内存管理困难
- 脚本调试体验不佳
- 学习成本高，用户需要编写代码

### 1.2 新需求：File Mode

从消费者导出的文件中读取消息，重新发送到 Kafka：
- 支持 JSON/CSV 格式（与消费者导出格式一致）
- 支持消息字段映射（key/value/timestamp 等）
- 支持批量发送和 TPS 控制
- 更直观的用户体验

## 2. 功能规格

### 2.1 支持的文件格式

#### JSON 格式（与消费者导出一致）
```json
[
  {
    "partition": 0,
    "offset": 12345,
    "timestamp": 1712345678901,
    "key": "order-001",
    "value": "{\"id\":1,\"status\":\"created\"}",
    "headers": {"source": "api"}
  }
]
```

#### CSV 格式（与消费者导出一致）
```csv
partition,offset,timestamp,key,value,headers
0,12345,1712345678901,order-001,"{""id"":1,""status"":""created""}","{""source"":""api""}"
```

### 2.2 字段映射

| 文件字段 | 说明 | 是否必需 |
|---------|------|---------|
| value | 消息内容 | 是 |
| key | 消息键 | 否（默认 null）|
| partition | 分区号 | 否（默认自动分配）|
| timestamp | 时间戳 | 否（默认当前时间）|
| headers | 消息头 | 否（默认空）|

#### Partition 处理策略

- **默认行为**: 不使用文件中的 partition，由 Kafka 自动分配分区
- **可选开启**: 用户可以通过勾选「使用文件中的 Partition」来按照文件中记录的 partition 发送消息
- **适用场景**: 
  - 保持消息顺序（当需要按原分区顺序发送时）
  - 特定分区测试（当需要向特定分区发送测试数据时）

### 2.3 发送策略

- **即时发送**: 一次性发送所有消息
- **TPS 控制**: 限制发送速率（1-10000 TPS）
- **定时发送**: 按计划发送（支持 Cron）

## 4. 技术设计

### 4.1 目录结构

```
kafkit/src/
├── pages/Producer/
│   ├── ProducerPage.tsx      # 主页面（移除 Script Mode 选项卡）
│   ├── SingleMode.tsx        # 单条模式
│   ├── BatchMode.tsx         # 批量模式
│   ├── ScheduledMode.tsx     # 定时模式
│   └── FileMode/             # 新增：文件模式
│       ├── index.tsx         # 主组件
│       ├── FileUploader.tsx  # 文件上传
│       ├── FieldMapper.tsx   # 字段映射
│       ├── MessagePreview.tsx # 消息预览
│       └── StrategyConfig.tsx # 发送策略
├── services/
│   └── fileParser.ts         # 文件解析服务
│       ├── parseJson.ts      # JSON 解析
│       └── parseCsv.ts       # CSV 解析
└── types/
    └── fileMode.ts           # 类型定义
```

### 4.2 核心类型

```typescript
interface FileMessage {
  value: string;
  key?: string;
  partition?: number;
  timestamp?: number;
  headers?: Record<string, string>;
}

interface FieldMapping {
  valueField: string;
  keyField?: string;
  partitionField?: string;
  timestampField?: string;
  headersField?: string;
}

interface FileSendConfig {
  mode: 'immediate' | 'tps' | 'cron';
  targetTPS?: number;
  cronExpression?: string;
  fieldMapping: FieldMapping;
}
```

### 4.3 文件解析服务

```typescript
class FileParser {
  parseJson(content: string): FileMessage[];
  parseCsv(content: string): FileMessage[];
  detectFields(sample: any): string[];  // 自动检测字段名
}
```

## 6. 注意事项

### 6.1 兼容性

- 必须与 Consumer 导出格式兼容
- 支持大文件（> 100MB）分块读取
- 支持中文文件名

### 6.2 性能

- 大文件使用 Web Worker 解析
- 虚拟滚动显示预览
- 批量发送避免内存溢出
