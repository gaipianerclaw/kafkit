## 安装

### macOS

1. 下载 `Kafkit_1.0.2_aarch64.dmg` (Apple Silicon) 或 `Kafkit_1.0.2_x64.dmg` (Intel)
2. 双击打开 DMG 文件
3. 将 Kafkit 拖到 Applications 文件夹
4. 首次运行时，在"系统设置 > 隐私与安全性"中允许打开

### Linux

**Ubuntu/Debian:**
```bash
sudo dpkg -i kafkit_1.0.2_amd64.deb
# 如果有依赖问题
sudo apt-get install -f
```

**Fedora/RHEL:**
```bash
sudo rpm -i kafkit-1.0.2-1.x86_64.rpm
```

### Windows

1. 下载 `Kafkit_1.0.2_x64-setup.exe`
2. 双击运行安装程序
3. 按向导完成安装

## 快速开始

### 1. 创建 Kafka 连接

1. 点击左上角连接选择器
2. 选择"新建连接"
3. 填写连接信息：
   - **名称**: 给连接起个名字（如"生产环境"）
   - **Bootstrap Servers**: Kafka 地址（如 `localhost:9092`）
   - **协议**: 选择安全协议（PLAINTEXT/SSL/SASL_PLAINTEXT/SASL_SSL）
   - **认证**: 根据协议选择认证方式
4. 点击"测试连接"验证
5. 点击"创建"保存

### 2. 查看 Topics

1. 从连接选择器选择一个连接
2. 点击左侧菜单"Topic 管理"
3. 查看该连接的所有 Topic 列表

### 3. 发送消息

1. 在 Topic 列表中点击 Topic 名称进入详情页
2. 点击右上角"发送消息"按钮
3. 选择发送模式（单条/批量/定时）
4. 填写消息内容并发送

### 4. 消费消息

1. 在 Topic 详情页点击"消费消息"按钮
2. 配置消费参数（起始位置、消费模式等）
3. 点击"开始消费"查看消息

## 连接管理

### 支持的协议

| 协议 | 说明 | 适用场景 |
|------|------|----------|
| PLAINTEXT | 无加密 | 内网、开发环境 |
| SSL | TLS 加密 | 生产环境 |
| SASL_PLAINTEXT | SASL 认证 | 需要用户名密码 |
| SASL_SSL | SASL + TLS | 高安全要求 |

### SSL 配置

需要以下证书文件（文本格式）：
- **CA 证书**: 服务器根证书
- **客户端证书** (可选): 双向认证时使用
- **客户端密钥** (可选): 双向认证时使用

⚠️ **注意**: 使用自签名证书时，请勾选"禁用主机名验证"

### SASL 认证

支持以下机制：
- **PLAIN**: 用户名/密码
- **SCRAM-SHA-256**: 安全的用户名/密码
- **SCRAM-SHA-512**: 更安全的用户名/密码
- **GSSAPI**: Kerberos 认证

### 导入/导出连接

- **导出**: 在连接管理页面，点击"导出"按钮可将所有连接导出为 JSON 文件
- **导入**: 点击"导入"按钮选择之前导出的 JSON 文件

## Topic 管理

### 新建 Topic

1. 在 Topics 页面点击"新建 Topic"
2. 填写：
   - **Topic 名称**: 支持字母、数字、点、下划线、连字符
   - **分区数**: 默认 1，可根据并发需求调整
   - **副本因子**: 默认 1，单节点集群只能设置为 1
3. 点击"创建"

⚠️ **注意**: 单节点 Kafka 集群的副本因子必须设置为 1

### 删除 Topic

⚠️ **警告**: 删除操作不可恢复！

1. 在 Topics 列表中找到要删除的 Topic
2. 点击右侧的删除图标
3. 在确认对话框中输入 Topic 名称
4. 点击"删除"确认

⚠️ 注意：Kafka 默认不允许删除 Topic，需要在 Kafka 配置中设置 `delete.topic.enable=true`

### Topic 详情

点击 Topic 名称查看详情：
- **统计卡片**: 分区总数、总消息数、平均副本因子、总副本数
- **分区详情**: Partition、Leader、Replicas、ISR、最早 Offset、最新 Offset、消息数
- **配置参数**: 查看和编辑 Topic 配置（如 retention.ms、cleanup.policy）

### 编辑 Topic 配置

1. 在 Topic 详情页点击"编辑配置"
2. 修改常用配置：
   - **retention.ms**: 数据保留时间（毫秒）
   - **cleanup.policy**: 清理策略（delete/compact）
   - **min.insync.replicas**: 最小同步副本数
   - **max.message.bytes**: 最大消息大小
3. 点击"保存"

## 消息生产

### 单条发送

1. 选择"单条"模式
2. 填写：
   - **Partition** (可选): 指定分区编号，留空则使用 key 的 hash
   - **Key** (可选): 消息键，用于分区选择
   - **消息内容**: 支持文本、JSON、CSV 格式
3. 点击"发送消息"

### 批量发送

1. 选择"批量"模式
2. 每行输入一条消息
3. 点击"批量发送"
4. 系统将逐条发送，失败的消息会显示错误

### 定时发送

1. 选择"定时"模式
2. 配置参数：
   - **批次大小**: 每次发送的消息数量
   - **发送间隔**: 每批次之间的间隔（毫秒）
   - **重复次数**: 发送轮数（0 表示无限循环）
3. 填写消息内容（每行一条）
4. 点击"开始定时发送"
5. 查看实时进度条和统计信息
6. 点击"停止"结束发送

## 消息消费

### 消费参数配置

在开始消费前，可以配置以下参数：

- **分区**: 选择特定分区或"所有分区"
- **起始位置**:
  - **Latest**: 从最新消息开始
  - **Earliest**: 从最早消息开始
  - **Timestamp**: 从指定时间戳开始
  - **Offset**: 从指定 Offset 开始
- **消费模式**:
  - **预览**: 在程序中显示消息
  - **文件**: 保存到本地文件
- **预览限制**: 最大显示消息数量（100/500/1000/无限制）

### 时间选择器

当选择"Timestamp"起始位置时：
- 快速选择：现在、1小时前、今天零点、昨天
- 自定义：选择具体日期时间

### 消息列表

消费开始后，消息以表格形式展示：
- **序号**: 消息序号
- **分区**: 所在分区
- **Offset**: 消息偏移量
- **时间戳**: 消息时间
- **Key**: 消息键
- **Value**: 消息内容预览

### 消息操作

- **展开详情**: 点击消息行查看完整信息（Headers、Size、Type 等）
- **复制内容**: 点击复制按钮复制消息内容
- **搜索**: 在右上角搜索框输入关键词搜索消息
- **大小写敏感**: 点击 "Aa" 按钮切换大小写敏感

### 导出消息

1. 消费消息后，点击"导出"按钮
2. 选择格式：JSON 或 CSV
3. 选择保存位置
4. 完成导出

### 停止消费

点击"停止"按钮停止消费。切换 Topic 时会自动停止当前消费。

## 消费者组管理

### 查看消费组列表

1. 点击左侧菜单"消费组"
2. 查看所有 Consumer Group：
   - **Group ID**: 消费组标识
   - **状态**: Stable（稳定）/ PreparingRebalance（准备重平衡）/ CompletingRebalance（完成重平衡）/ Dead（死亡）/ Empty（空）
   - **成员数**: 消费组中的消费者数量
   - **协调器**: 负责该消费组的 Broker

### 查看消费延迟(Lag)

1. 在消费组列表中点击某个 Group
2. 右侧面板显示该 Group 的 Lag 信息：
   - **Topic**: Topic 名称
   - **Partition**: 分区编号
   - **Current Offset**: 当前消费位置
   - **Log End Offset**: 最新消息位置
   - **Lag**: 未消费消息数量
   - **状态**: 正常/警告（Lag > 1000）

3. 顶部显示总 Lag 数量和成员详情

### 偏移量重置

⚠️ **注意**: 消费组必须处于 Empty 状态才能重置偏移量

1. 在 Lag 详情面板中点击"重置偏移量"按钮
2. 选择重置策略：
   - **Earliest**: 重置到最早消息
   - **Latest**: 重置到最新消息
   - **Offset**: 重置到指定 Offset
3. 选择作用范围：
   - **整个 Topic**: 重置该 Topic 的所有分区
   - **单个分区**: 仅重置选中的分区
4. 点击"确认重置"

⚠️ **警告**: 偏移量重置会影响消费进度，请谨慎操作！

## 常见问题

### Q: 连接测试失败怎么办？

检查以下几点：
1. Kafka 地址和端口是否正确
2. 网络是否可达（尝试 `telnet host port`）
3. 安全协议和认证信息是否正确
4. 防火墙是否允许访问
5. 如果使用 SSL/SASL_SSL，证书和认证信息是否正确

### Q: 无法删除 Topic？

Kafka 默认禁止删除 Topic。需要在 Kafka 配置 (`server.properties`) 中添加：
```properties
delete.topic.enable=true
```
然后重启 Kafka。

### Q: 连接后看不到 Topics？

可能原因：
1. 该 Kafka 集群确实没有 Topics
2. 当前用户没有权限查看 Topics
3. 连接配置有误（如错误的 SASL 认证）
4. 安全组或防火墙限制

### Q: 消费组显示 Empty 状态，看不到 Lag 信息？

这是正常行为。Lag 信息只在消费组有活跃消费者和已提交偏移量时才会显示。可能原因：
1. 消费组还没有开始消费
2. 没有提交过偏移量
3. 消费组已过期被清理

### Q: 使用自签名证书连接失败？

在连接配置的 SSL 证书部分，勾选"禁用主机名验证"选项。

### Q: 发送消息失败？

检查以下几点：
1. Topic 是否存在
2. 是否有写入权限
3. 消息大小是否超过限制（默认 1MB）
4. 如果是压缩消息，确保 Kafka 支持相应压缩格式

### Q: 消费消息时显示 "NotImplemented" 错误？

这通常是因为 Kafka 消息使用了压缩（gzip/lz4/zstd），而客户端不支持。本版本已添加压缩支持，如仍有问题请检查：
1. Kafka 版本兼容性
2. 网络连接稳定性
