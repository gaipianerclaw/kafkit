# Kafkit v1.0.3 优化路线图

> **版本目标**: 性能优化、稳定性提升、用户体验改进
> **预计工期**: 2-3 周
> **优先级**: P0(必须) > P1(重要) > P2(建议)

---

## 一、性能优化

### 1.1 大数据量消费优化 (P0)

**问题**: 当前消费大量消息时（>10000条），UI 会出现卡顿甚至崩溃

**优化方案**:
- [ ] 实现虚拟滚动（Virtual Scrolling）替代全量渲染
  - 使用 `react-window` 或 `@tanstack/react-virtual`
  - 只渲染可视区域的消息，大幅提升性能
- [ ] 消息分页加载，替代无限追加
  - 设置内存上限（如最多保留 5000 条）
  - 超出时提示用户"消息数量过多，建议导出到文件查看"
- [ ] 大数据量时自动切换到文件模式
  - 当消息数超过阈值（如 1000 条）提示切换

**验收标准**:
- 消费 10万条消息不卡顿
- 内存占用稳定在 500MB 以内

---

### 1.2 连接池优化 (P1)

**问题**: 当前每个操作都新建连接，连接未复用

**优化方案**:
- [ ] 实现连接池管理
  - 连接空闲超时（如 5 分钟）自动关闭
  - 最大连接数限制
  - 连接健康检查
- [ ] AdminClient 连接复用
  - Topic 列表、创建、删除等操作复用同一 AdminClient
- [ ] Consumer/Producer 延迟创建
  - 首次使用时才创建
  - 使用完后延迟关闭（避免频繁创建销毁）

**验收标准**:
- 连续操作不创建多余连接
- 连接数在合理范围内（< 10 个）

---

### 1.3 Topic 列表加载优化 (P1)

**问题**: Topic 数量多（>1000）时加载缓慢

**优化方案**:
- [ ] 分页加载或虚拟列表
- [ ] 搜索过滤（客户端侧）
- [ ] 缓存 Topic 元数据，增量更新

---

## 二、稳定性提升

### 2.1 错误处理完善 (P0)

**问题**: 部分错误场景处理不完善，用户体验差

**改进项**:
- [ ] 连接断开的优雅处理
  - 自动重连机制（3次重试）
  - 断开时显示状态提示
  - 重连成功后恢复操作
- [ ] 超时处理
  - 消费超时（默认 30 秒无消息则提示）
  - 发送超时处理
- [ ] 错误信息国际化
  - 将 rdkafka 的错误码映射为友好提示
  - 提供解决建议（如"连接超时，请检查网络"）

**常见错误映射**:
```
NetworkException -> "网络连接失败，请检查地址和端口"
UnknownTopicOrPartition -> "Topic 不存在或暂无数据"
NotLeaderForPartition -> "正在选举 Leader，请稍后重试"
AuthorizationFailed -> "认证失败，请检查用户名密码"
SSLHandshakeFailed -> "SSL 握手失败，请检查证书配置"
```

---

### 2.2 内存泄漏排查 (P1)

**检查项**:
- [ ] Consumer 关闭时清理资源
- [ ] 定时器/事件监听器的清理
- [ ] React 组件卸载时的清理
- [ ] Rust 端的资源释放

---

### 2.3 数据一致性保障 (P2)

**问题**: 消费过程中可能出现数据丢失或重复

**改进项**:
- [ ] 消费确认机制（手动 commit offset）
- [ ] 消费进度保存（支持中断后恢复）
- [ ] 重复消息去重（基于 offset）

## 三、用户体验改进

- [ ] 支持更多消息格式

### 3.2 快捷操作 (P1)

**功能需求**:
- [ ] 快捷键支持
  ```
  Cmd/Ctrl + K: 快速跳转（类似 VS Code Command Palette）
  Cmd/Ctrl + R: 刷新当前页面
  Cmd/Ctrl + Enter: 发送消息
  Esc: 关闭弹窗/停止消费
  ```
- [ ] 最近访问的 Topic 列表
- [ ] 收藏功能（常用 Topic）

---

### 3.3 搜索增强 (P1)

**功能需求**:
- [ ] 全局搜索
  - 跨 Topic 搜索消息
  - 搜索结果跳转
- [ ] 高级搜索语法
  ```
  topic:order partition:0 key:user123
  timestamp:>2024-01-01 value:contains("error")
  ```
- [ ] 搜索历史记录

---

### 3.4 UI 细节优化 (P2)

**改进项**:
- [ ] 加载状态统一
  - 骨架屏替代 Spin 动画
  - 渐进式加载提示

  - 更友好的空状态插画和提示
- [ ] 消息时间显示

- [ ] 自动刷新机制

## 六、Bug 修复（已知问题）

### 6.1 高优先级

- [ ] **DMG 打包偶发失败** - 需要排查 bundle_dmg.sh 失败原因
- [ ] **消费组偏移量重置未实际生效** - 目前仅计算目标 offset，需要实现真正的 seek 和 commit
- [ ] **大消息显示截断** - 超过一定大小的消息无法完整显示

### 6.2 中优先级

- [ ] **时间戳选择器时区问题** - 需要明确区分本地时间和 UTC
- [ ] **连接配置验证不完整** - SSL 证书格式校验、必填项校验
- [ ] **定时发送停止后状态未完全重置** - 需要检查清理逻辑

### 6.3 低优先级

- [ ] **窗口大小记忆** - 重启后恢复上次窗口大小
- [ ] **列宽调整记忆** - 表格列宽调整后记忆偏好

| 风险 | 概率 | 影响 | 应对措施 |
