# Kafkit 浏览器预览模式

## 🚀 快速开始

浏览器预览模式已启用！你现在可以在浏览器中测试 Kafkit 的界面和交互。

### 访问地址
```
http://localhost:1420/
```

## 📋 预览模式说明

### ✅ 可用的功能

| 功能 | 状态 | 说明 |
|------|------|------|
| 界面布局 | ✅ 完全可用 | 所有 UI 组件正常显示 |
| 连接管理 | ✅ Mock 数据 | 可创建/编辑/删除连接 |
| Topic 列表 | ✅ Mock 数据 | 显示模拟 Topic 数据 |
| Topic 详情 | ✅ Mock 数据 | 显示分区、配置信息 |
| 消息消费 | ✅ Mock 数据 | 模拟实时消息流 |
| 消息生产 | ✅ Mock 数据 | 模拟发送消息 |
| Consumer Group | ✅ Mock 数据 | 显示模拟 Group 数据 |

### ⚠️ 限制

由于是浏览器预览（非桌面应用），以下功能使用模拟数据：
- Kafka 连接为模拟连接
- Topic 数据为模拟数据
- 消息为自动生成的模拟消息

**真实 Kafka 连接需要在桌面应用中运行**

## 🎯 测试场景

### 场景 1: 创建 EC2 Kafka 连接

1. 打开 http://localhost:1420/
2. 点击 **"创建第一个连接"** 或左上角连接下拉菜单 → "新建连接"
3. 填写信息：
   - **连接名称**: `EC2 Kafka`
   - **Bootstrap Servers**: `ec2-dmz-kafka-01:9092`
   - **安全协议**: `PLAINTEXT`
   - **认证方式**: `无认证`
4. 点击 **"测试连接"**
   - 预期结果：✅ 连接成功！发现 4 个 Topics
5. 点击 **"创建"**

### 场景 2: 浏览 Topics

1. 在左侧导航选择 **"Topics"**
2. 查看 Topic 列表：
   - user-events (3 分区)
   - order-events (6 分区)
   - system-logs (1 分区)
3. 点击搜索框，输入 "user" 过滤
4. 点击 **user-events** 查看详情
5. 查看分区信息、副本、配置参数

### 场景 3: 消费消息

1. 在 Topic 列表点击 user-events 的 **⬇️ 消费** 按钮
2. 点击 **"开始"** 按钮
3. 观察实时消息流（每 2 秒自动推送一条模拟消息）
4. 查看消息内容（JSON 格式化显示）
5. 点击 **"暂停"** 停止消费
6. 点击 **"导出"** 下载消息为 JSON 文件

### 场景 4: 发送消息

1. 在 Topic 列表点击 user-events 的 **⬆️ 发送** 按钮
2. 选择 **JSON** 格式
3. 点击 **"加载示例"**
4. 编辑消息内容：
   ```json
   {
     "id": 999,
     "userId": "test-user",
     "action": "login",
     "timestamp": "2026-03-13T23:30:00Z"
   }
   ```
5. 点击 **"发送"**
6. 查看发送成功提示

### 场景 5: 批量发送

1. 在发送页面切换到 **"批量"** 模式
2. 输入多条消息（每行一条）：
   ```
   {"id": 1, "msg": "message 1"}
   {"id": 2, "msg": "message 2"}
   {"id": 3, "msg": "message 3"}
   ```
3. 点击 **"批量发送"**
4. 查看发送结果统计

### 场景 6: 查看 Consumer Groups

1. 在左侧导航选择 **"Consumer Groups"**
2. 查看 Group 列表：
   - user-service (3 members, Stable)
   - order-service (2 members, Stable)
   - analytics-consumer (0 members, Empty)
3. 点击 **user-service**
4. 查看消费 Lag：
   - user-events 分区 0: Lag 500
   - user-events 分区 1: Lag 200
   - order-events 分区 0: Lag 3000

## 🔧 故障排除

### 页面空白
1. 检查 http://localhost:1420/ 是否可访问
2. 查看浏览器控制台（F12）是否有错误
3. 刷新页面重试

### 接口错误
如果看到 "Cannot read properties of undefined" 错误：
- ✅ 已修复 - 添加了 Mock 服务层
- 刷新页面即可

## 🖥️ 构建桌面应用

浏览器预览适合 UI 测试，真实 Kafka 连接需要桌面应用：

```bash
# 安装 Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 构建桌面应用
cd kafkit
npm run tauri:build

# 运行桌面应用
./src-tauri/target/release/kafkit
```
